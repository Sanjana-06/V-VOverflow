Do we need Docker ?

We develop applications for the railway domain, it is quite different than the one you usually see, like e-commerce, banking or video streaming platforms. Let us dig deeper to understand this aspect.

There are many good things about Docker. It packs, ships, and runs applications as a lightweight, portable, and self-sufficient containerization tool. Docker is great for businesses of all sizes. When you are working on a piece of code in a small team, it eliminates the “but it works on my machine” problem.

But in a server you will rarely have “but it works on my machine” issue and also what if . . .

  • Speed is my primary concern ?
  • I want to priotize security
  • I am looking for easily managed technology
  • I only have a few nodes to manage

BAUDET Nicolas

BAUDET Nicolas, is a Senior expert DIS working for DMS, at Villeurbanne. During his recent visit to Bangalore, I had a good discussion on this topic with him. An extract of the discussion is presented below :

Container technology can be beneficial even on a single node and does not need Kubernetes. However, using many containers quickly brings the need for a container orchestrator (=Kubernetes) even on a single node, adding more complexity and specific skillset while using only a subpart of Kubernetes potential. Kubernetes can reach full potential, when the number of nodes start to get more than 10, in general the more the nodes the better. When to containerize is a decision to be taken by weighing betwen application needs and the complexity it brings in.

One has to look into the conditions these technologies can be beneficial in Alstom context. Tek Bytes will present more details about this in the up comming series.
